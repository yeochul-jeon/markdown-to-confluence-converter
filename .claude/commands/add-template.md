# /add-template — templates.js에 새 문서 템플릿 추가

`js/templates.js`에 새로운 문서 템플릿을 추가합니다.

## 인자

추가할 템플릿 이름/종류: $ARGUMENTS

## 실행 절차

### 1. 기존 템플릿 구조 확인

`js/templates.js`의 기존 4개 템플릿 구조를 확인합니다:

```javascript
{
  id: 'kebab-case-id',       // 고유 ID (kebab-case)
  icon: '&#128196;',          // HTML 엔티티 이모지
  name: '한국어 이름',         // 카드에 표시되는 이름
  desc: '한 줄 설명',          // 카드 하단 설명
  content: `마크다운 내용`     // 실제 템플릿 (백틱 템플릿 리터럴)
}
```

기존 ID: `basic-doc`, `table-doc`, `api-doc`, `meeting-note`

### 2. 새 템플릿 작성

`$ARGUMENTS`에 맞는 실용적인 마크다운 템플릿을 작성합니다:

- **분량**: 15~40줄 (너무 짧지도 길지도 않게)
- **요소 활용**: 제목, 리스트, 테이블, 코드 블록, 인용 등 다양한 마크다운 요소 포함
- **실용성**: 실제 업무에서 바로 쓸 수 있는 구조
- **한국어**: 예시 텍스트는 한국어로 작성

아이콘은 템플릿 성격에 맞는 HTML 엔티티 이모지를 선택합니다.

### 3. templates.js 수정

`list` 배열의 마지막 항목 뒤에 새 템플릿 객체를 추가합니다.
기존 코드 스타일(들여쓰기, 따옴표 등)을 정확히 따릅니다.

### 4. 변환 테스트

새 템플릿의 `content`를 파이프로 변환하여 Confluence 마크업이 올바르게 생성되는지 확인합니다:

```bash
echo "<템플릿 content>" | node cli.js --stdout
```

변환 결과에 문제가 있으면 content를 조정합니다.

### 5. 브라우저 확인 안내

사용자에게 다음 사항을 안내합니다:
- `index.html`을 브라우저에서 열어 새 템플릿 카드가 표시되는지 확인
- 카드 클릭 시 에디터에 마크다운이 로드되는지 확인
- 변환 버튼 클릭 시 Confluence 마크업이 올바르게 생성되는지 확인
